### Summary
This PR transitions BetterBags from a dynamic item button pool to a static `bag:slot` button caching architecture. It introduces a permanent 1-to-1 physical slot to button registry to completely eliminate dynamic frame pool mutations and combat lockdown taint. Redundant and dangerous ID mutations (like `SetBagID` during `SetItemFromData`, `SetFreeSlots`, and `ClearItem`) have been removed since the ID is permanently bound to the physical button upon creation. It also updates bag views and grid views to use the static item buttons correctly (including fetching physical buttons for merged free slots using `slotInfo.freeSlotKeys`).

### Motivation
The previous architecture dynamically mapped `bag` and `slot` to recycled item buttons. This required modifying the secure attribute `"bagid"` using the hidden parent frame workaround during combat to avoid taint blocks. Moving to a permanent static `bag:slot` item button architecture completely resolves these secure taint issues by avoiding `SetBagID` mutations during combat entirely. It also provides a significant performance boost by eliminating the overhead of pool management and frame recycling.
